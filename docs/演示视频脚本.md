## 3:30 – 4:30 数字人动画与语音交互

**画面**：特写 Canvas 数字人区域 + 聊天区

### 场景 E：数字人表情状态（3:30 – 4:00）

**操作**：
1. 提一个问题，展示数字人状态变化：
   - 发送问题 → 数字人进入「思考」状态（歪头、蹙眉）
   - 收到第一个 token → 数字人进入「说话」状态（嘴巴开合、腮红、眨眼）
   - 回答完成 → 数字人恢复「待机」状态
2. 强调三种状态的细节差异

**旁白要点**：
- 「我们的数字人不是 GIF 或视频，而是用 Canvas 2D 画布实时绘制的矢量角色。」
- 「思考时眉毛上挑、头部微倾、嘴巴歪向一侧。说话时嘴巴按正弦波开合，加上随机扰动模拟真人说话的节奏感，还有腮红效果。」
- 「全程由 SSE 流的 expression 事件驱动状态切换，与后端生成过程实时同步。」
- 「数字人动画可以独立开关，深色模式下也能正常显示。」

### 场景 F：语音闭环（4:00 – 4:30）

**操作**：
1. **按住** 麦克风按钮，说：「拈花湾有什么好玩的？」
2. 松手 → 显示识别的文字 → 自动发送 → 流式回答 → 自动 TTS 语音播放

**旁白要点**：
- 「支持按住说话语音输入，前端用 MediaRecorder 录音，后端用 faster-whisper 做中文识别。」
- 「识别后自动填入并发送，回答完成后自动用 edge-tts 合成语音播放。」
- 「形成『说 → 识别 → 问答 → 播报』的完整语音闭环，解放双手。」
- 「TTS 支持 4 种音色切换（晓晓、云希、晓伊、云健），语速也可调节。」
